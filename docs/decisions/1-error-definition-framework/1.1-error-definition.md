# 1.1 Error Definition

[← Kembali ke 1. Error Definition Framework](./README.md)

## Tujuan

Menetapkan bentuk metadata standar yang melekat pada setiap enum case error.

## Keputusan

Metadata error disimpan menggunakan PHP Attribute pada enum case.

Kontrak lengkap `ErrorDefinition` tersedia pada [ErrorDefinition Reference](../../reference/error-definition.md). Konsep field metadata dijelaskan pada [Error Definition](../../concepts/error-definition.md).

## Alasan Menggunakan PHP Attribute

PHP Attribute dipilih karena metadata dapat ditempatkan langsung pada enum case yang memiliki error code terkait.

Pendekatan ini:

- menempatkan kode dan definisi error pada satu lokasi;
- mencegah duplikasi metadata;
- dapat dibaca menggunakan PHP Reflection;
- dapat digunakan sebagai sumber generator TypeScript dan JSON;
- dapat digunakan oleh validation, exception, registry, dan dokumentasi.

## Keputusan Terkait

- Error code menggunakan PHP Backed Enum.
- `httpStatus` wajib ditentukan secara eksplisit.
- `message` menjadi default message untuk exception dan validation.
- `ErrorDefinition` hanya menyimpan metadata runtime.
- Knowledge Base, owner, SLA, escalation, dan registry metadata berada di luar kontrak runtime.

## Konsekuensi

- Metadata error memiliki satu sumber kebenaran.
- Consumer tidak perlu menyimpan definisi metadata di banyak tempat.
- Reader dapat mengambil metadata menggunakan Reflection.
- Resource lain dapat dihasilkan dari definisi yang sama.
