# 1.1 Error Definition

[← Kembali ke 1. Error Definition Framework](./README.md)

## Tujuan

`ErrorDefinition` merupakan metadata standar yang melekat pada setiap error code. Metadata ini menjadi **Single Source of Truth** untuk menentukan representasi error pada exception, validation, dan proses pembuatan catalog.

## Keputusan

Metadata disimpan menggunakan **PHP Attribute** yang ditempelkan pada setiap enum case.

```php
#[Attribute(Attribute::TARGET_CLASS_CONSTANT)]
final readonly class ErrorDefinition
{
    public function __construct(
        public string $message,
        public ErrorCategory $category,
        public int $httpStatus,
        public ErrorSeverity $severity = ErrorSeverity::MEDIUM,
        public bool $retryable = false,
    ) {}
}
```

## Contoh Penggunaan

```php
enum SuratMasukError: string
{
    #[ErrorDefinition(
        message: 'Nomor surat wajib diisi.',
        category: ErrorCategory::VALIDATION,
        httpStatus: 422,
        severity: ErrorSeverity::LOW,
    )]
    case NOMOR_SURAT_REQUIRED = 'SM-VAL-001';

    #[ErrorDefinition(
        message: 'Surat sudah disetujui dan tidak dapat diubah.',
        category: ErrorCategory::WORKFLOW,
        httpStatus: 409,
    )]
    case DRAFT_LOCKED = 'SM-WF-001';
}
```

## Alasan Menggunakan PHP Attribute

PHP Attribute dipilih karena metadata dapat ditempatkan langsung pada enum case yang memiliki error code terkait.

Pendekatan ini:

- menempatkan kode dan definisi error pada satu lokasi;
- mencegah duplikasi metadata;
- dapat dibaca menggunakan PHP Reflection;
- dapat digunakan sebagai sumber pembuatan TypeScript dan JSON;
- menjadi dasar untuk validation, exception, registry, dan dokumentasi.

## Field Metadata

| Field | Keterangan |
|-------|------------|
| `message` | Pesan default yang aman ditampilkan kepada pengguna. |
| `category` | Kategori error, seperti Validation, Workflow, Business Rule, atau Integration. |
| `httpStatus` | HTTP status yang dikembalikan oleh API. Field ini wajib ditentukan agar semantik error eksplisit. |
| `severity` | Tingkat dampak error untuk logging, monitoring, dan analisis. Nilai default adalah `MEDIUM`. |
| `retryable` | Menentukan apakah operasi dapat dicoba kembali. Nilai default adalah `false`. |

## Metadata yang Tidak Disimpan

Field berikut tidak menjadi bagian dari `ErrorDefinition` karena merupakan concern komponen atau tahap berikutnya:

- Knowledge Base (KB);
- Owner;
- SLA;
- Escalation;
- Registry metadata;
- Module;
- Error Code.

Error code tidak disimpan dalam attribute karena sudah berasal dari value pada Backed Enum.

## Konsekuensi

- Metadata error berada dekat dengan error code.
- `httpStatus` selalu ditentukan secara eksplisit.
- `message` menjadi default message untuk exception dan validation.
- `ErrorDefinition` hanya menyimpan metadata yang dibutuhkan aplikasi pada runtime.
- Metadata tambahan seperti owner, SLA, dan knowledge base dapat dikembangkan pada komponen terpisah tanpa mengubah kontrak runtime ini.
