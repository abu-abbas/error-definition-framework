# 1.1 Error Definition

[← Kembali ke 1. Error Definition Framework](./README.md)

> **TL;DR:** kode, pesan, dan karakteristik dasar sebuah error ditulis satu kali agar semua komponen aplikasi menggunakan definisi yang sama.

## Tujuan

Menetapkan bentuk metadata standar yang melekat pada setiap enum case error.

## Keputusan

Metadata error disimpan menggunakan PHP Attribute pada enum case.

`httpStatus` wajib ditentukan secara eksplisit. `message` menjadi pesan default untuk exception dan validation. Category, severity, dan retryability menjadi metadata typed yang dapat digunakan oleh seluruh consumer.

## Alasan

PHP Attribute menempatkan metadata langsung di dekat error code yang dijelaskannya. Pendekatan ini mencegah definisi tersebar di validation, exception, response, dan generator.

Attribute dapat dibaca menggunakan Reflection sehingga komponen lain dapat memperoleh metadata tanpa membuat registry manual. Definisi yang sama juga dapat menjadi sumber pembuatan TypeScript dan JSON.

## Keputusan Terkait

- Error code menggunakan PHP Backed Enum.
- Kontrak field tersedia pada [ErrorDefinition Reference](../../reference/error-definition.md).
- Makna metadata dijelaskan pada [Error Definition](../../concepts/error-definition.md).
- Batas metadata runtime dan registry ditetapkan pada [Decision 1.1.1](./1.1.1-batas-metadata-runtime-dan-registry.md).

## Konsekuensi

- Metadata error memiliki satu sumber kebenaran.
- Consumer tidak perlu mendefinisikan metadata di banyak tempat.
- Reader dapat mengambil metadata menggunakan Reflection.
- Resource lain dapat dihasilkan dari definisi yang sama.
- Perubahan kontrak attribute memengaruhi seluruh consumer dan harus diperlakukan sebagai perubahan framework.
