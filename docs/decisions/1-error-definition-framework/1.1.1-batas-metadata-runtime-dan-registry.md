# 1.1.1 Batas Metadata Runtime dan Registry

[← Kembali ke 1. Error Definition Framework](./README.md)

> **TL;DR:** aplikasi menyimpan informasi yang diperlukan untuk menghasilkan error, sedangkan registry menyimpan informasi organisasi yang dapat berubah tanpa mengubah aplikasi.

## Tujuan

Menetapkan batas antara metadata yang menjadi bagian dari source code aplikasi dan metadata yang dikelola oleh Central Error Registry.

## Keputusan

`ErrorDefinition` hanya menyimpan metadata yang diperlukan ketika aplikasi berjalan:

- message;
- category;
- HTTP status;
- severity;
- retryable.

Knowledge Base, owner, SLA, escalation, module registry, dan metadata operasional lain tidak menjadi bagian dari kontrak runtime. Informasi tersebut dikelola oleh Central Error Registry atau sistem operasional terkait dengan menggunakan error code sebagai penghubung.

## Alasan

Metadata runtime menentukan perilaku langsung aplikasi dan harus tersedia bersama source code yang menghasilkan error. Metadata organisasional memiliki lifecycle berbeda: owner dapat berganti, SLA dapat diperbarui, dan artikel Knowledge Base dapat berubah tanpa release aplikasi.

Menyatukan keduanya akan membuat perubahan operasional membutuhkan perubahan source code dan deployment. Memisahkannya menjaga aplikasi tetap stabil sekaligus memungkinkan registry berkembang secara independen.

## Keputusan Terkait

- Laravel tetap menjadi source of truth untuk error code dan metadata runtime.
- Central Error Registry menjadi consumer, bukan dependency runtime aplikasi.
- Error code menjadi key yang menghubungkan definisi aplikasi dengan metadata registry.
- Konsep batas metadata dijelaskan pada [Error Definition](../../concepts/error-definition.md).

## Konsekuensi

- Aplikasi tetap dapat menghasilkan response error tanpa Central Error Registry.
- Perubahan owner, SLA, escalation, atau Knowledge Base tidak memerlukan deployment aplikasi.
- Registry harus menjaga relasi metadata organisasional berdasarkan error code yang stabil.
- Generator catalog perlu membedakan field runtime dari enrichment milik registry.
