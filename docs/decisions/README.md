# Design Decisions

[← Kembali ke documentation index](../README.md)

Dokumen pada bagian ini mencatat alasan dan konsekuensi keputusan desain framework.

## Menulis Decision Doc

Semua decision doc baru dimulai dari [template ADR](./_template.md).

- Gunakan urutan `Tujuan → Keputusan → Alasan → Keputusan Terkait → Konsekuensi`.
- Tulis TL;DR non-teknis dalam satu kalimat.
- Sertakan contoh dan counter-example jika keputusan membutuhkan judgment.
- Tempatkan tutorial, contoh cara pakai, dan struktur folder pada `guides/`.
- Tambahkan istilah teknis lintas dokumen ke [glossary](../concepts/istilah.md).
- Tempatkan bentuk API dan kontrak class pada `reference/`.
