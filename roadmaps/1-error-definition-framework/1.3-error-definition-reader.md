# 1.3 ErrorDefinitionReader

## Tujuan

`ErrorDefinitionReader` bertugas membaca metadata error dari sebuah enum case yang mengimplementasikan `ErrorCode`, kemudian mengubahnya menjadi object yang siap digunakan oleh consumer.

Framework ini menjadikan enum sebagai sumber kode error dan PHP Attribute sebagai sumber metadata, sehingga seluruh informasi error dapat diperoleh dari satu tempat (Single Source of Truth).

---

# Responsibility

`ErrorDefinitionReader` hanya memiliki satu tanggung jawab:

> Membaca satu enum case dan mengembalikan metadata error yang telah di-resolve.

Contoh penggunaan:

```php
$definition = $reader->read(
    SuratMasukError::DRAFT_LOCKED
);
```

Reader **tidak** bertanggung jawab terhadap:

* scan folder/module
* menemukan seluruh enum error
* generate `error-catalog.json`
* generate `error-codes.ts`
* logging
* registry
* validasi duplicate code
* validasi struktur aplikasi

Seluruh kebutuhan tersebut akan ditangani pada roadmap berikutnya.

---

# Input

Method `read()` menerima enum yang mengimplementasikan marker interface `ErrorCode` sekaligus merupakan `BackedEnum`.

```php
public function read(
    ErrorCode&BackedEnum $error
): ResolvedErrorDefinition
```

Marker interface digunakan agar hanya enum error yang dapat diterima oleh reader.

Contoh:

```php
enum SuratMasukError: string implements ErrorCode
{
    #[ErrorDefinition(...)]
    case DRAFT_LOCKED = 'SM-WF-001';
}
```

Dengan pendekatan ini, enum lain seperti `Status`, `Role`, atau `DocumentType` tidak dapat diproses secara tidak sengaja.

---

# Output

`read()` mengembalikan object `ResolvedErrorDefinition`.

```php
final readonly class ResolvedErrorDefinition
{
    public function __construct(
        public string $code,
        public string $message,
        public ErrorCategory $category,
        public int $httpStatus,
        public ErrorSeverity $severity,
        public bool $retryable,
    ) {}
}
```

Object ini merupakan hasil penggabungan antara:

* kode error dari enum
* metadata dari `ErrorDefinition`

Consumer tidak perlu lagi membawa enum maupun attribute secara terpisah.

---

# Missing Attribute

Apabila enum yang mengimplementasikan `ErrorCode` tidak memiliki attribute `ErrorDefinition`, maka reader harus melempar exception.

```php
throw new MissingErrorDefinitionException(...);
```

Alasan:

* merupakan kesalahan konfigurasi programmer
* bukan business error
* menghindari `null` checking pada seluruh consumer
* menjaga kontrak `read()` tetap konsisten

---

# Reflection

Metadata dibaca menggunakan PHP Reflection.

Alur kerjanya:

1. menerima enum case
2. membaca `ReflectionEnumUnitCase`
3. mengambil attribute `ErrorDefinition`
4. membangun `ResolvedErrorDefinition`
5. mengembalikan hasilnya

Reflection hanya digunakan sebagai mekanisme pembacaan metadata.

---

# Cache

Reader menggunakan cache internal berbasis memory.

```php
private array $cache = [];
```

Cache menyimpan object `ResolvedErrorDefinition` yang sudah selesai di-resolve, bukan object reflection.

Contoh:

```text
SuratMasukError::DRAFT_LOCKED
    => ResolvedErrorDefinition(...)
```

Keuntungan:

* reflection hanya dilakukan sekali
* consumer selalu memperoleh object siap pakai
* tidak bergantung pada Laravel Cache maupun Redis
* aman digunakan sebagai singleton

---

# Kompatibilitas dengan Octane

Pendekatan cache internal tetap aman digunakan pada Laravel Octane karena metadata error bersifat immutable.

Syaratnya:

* cache hanya berisi metadata statis
* tidak menyimpan request
* tidak menyimpan user
* tidak menyimpan locale
* tidak menyimpan context runtime

Perubahan source code tetap mengikuti mekanisme reload worker saat deployment.

---

# Hal yang Tidak Disimpan

`ResolvedErrorDefinition` tidak menyimpan:

* enum source
* `BackedEnum`
* `Reflection*`
* `ReflectionAttribute`
* object hasil reflection lainnya

Object ini hanya berisi data hasil resolve sehingga mudah digunakan, diuji, maupun diserialisasi.

---

# Keputusan

* `ErrorDefinitionReader` memiliki satu tanggung jawab: membaca satu enum case.
* Input menggunakan `ErrorCode & BackedEnum`.
* Output menggunakan `ResolvedErrorDefinition`.
* Attribute yang hilang menghasilkan `MissingErrorDefinitionException`.
* Metadata dibaca menggunakan PHP Reflection.
* Hasil resolve disimpan pada cache internal.
* Cache hanya menyimpan metadata immutable.
* `ResolvedErrorDefinition` tidak menyimpan enum maupun object reflection.

---

# Catatan

Pembahasan berikut **tidak termasuk ruang lingkup** `ErrorDefinitionReader` dan akan dibahas pada roadmap selanjutnya:

* `ApplicationException`
* Integrasi `FormRequest`
* Standar JSON Response
* Logging & Context
* Error Linter
* Generator `error-codes.ts`
* Generator `error-catalog.json`
* Central Error Registry
